The primary technology stack should be standardized across all projects to ensure consistency and comparability. Use Java 21 as the programming language and Spring Boot 3.x as the core application framework. Adopt Maven for build management and PostgreSQL as the default relational database. Use Spring Data JPA and Hibernate for persistence, Flyway for database migrations, OpenAPI/Swagger for API documentation, JUnit 5, Mockito, Testcontainers, and WireMock for testing. Use Redis for caching and distributed coordination, RabbitMQ for queue-based messaging and Pub/Sub scenarios, Apache Kafka for event-driven architecture and streaming use cases, MongoDB for document-oriented persistence, Cassandra for large-scale event and time-series storage, DynamoDB for cloud-native NoSQL demonstrations, and OpenSearch for search and indexing capabilities. Implement resilience using Resilience4j, observability using Micrometer, Prometheus, Grafana, ELK/OpenSearch logging stack, and OpenTelemetry-based distributed tracing. Containerization should be based on Docker and Docker Compose, while orchestration should use Kubernetes and Helm. Cloud-native implementations should target AWS using ECS, EKS, RDS, ElastiCache, SQS, SNS, MSK, CloudWatch, X-Ray, S3, IAM, Secrets Manager, and Lambda where appropriate. CI/CD examples should use GitHub Actions as the primary pipeline technology. All projects should follow clean coding principles, Domain-Driven Design where applicable, RESTful API standards, Twelve-Factor App principles, secure-by-default practices, infrastructure-as-code concepts, and production-grade operational patterns commonly used in enterprise-scale distributed systems.
